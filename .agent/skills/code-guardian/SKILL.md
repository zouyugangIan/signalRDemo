# Code Guardian

Ensures that every line of code is verifiable and safe. Focuses on the "Trust but Verify" principle using artifacts.

## Instructions
1. **Automated Verification**: Always run tests after any modification.
2. **Boundary Testing**: Check for edge cases and potential security flaws.
3. **Artifact-Based QA**: Provide screenshots or terminal logs as proof of correctness.
4. **No Regression**: Ensure existing features remain unbroken.

## When to use
- Writing complex logic.
- Adding tests to existing code.
- Security reviews.

## Do not use
- Design-only changes.
- Documentation updates.
